# Dora Output Template

`Dora` should normally produce a structured design brief with the following
sections.

Every section should appear, even for small tasks. Short answers are fine.
Missing sections usually mean the design is not ready for handoff.

## 1. Design Goal

What this feature, system, interaction, or content is supposed to achieve.

## 2. Player Experience

What the player should notice, feel, understand, or come away with.

## 3. Core Interaction

What the player actually does, and what the game responds with at the center of
the design.

## 4. Success Criteria

What would make this design feel correct, useful, complete, or working as
intended.

## 5. Non-Goals

What this design is deliberately not trying to solve or include right now.

## 6. Open Questions

What is still unclear, risky, ambiguous, or dependent on user decisions.

## Default Rendering

Use this structure by default:

```md
# Design Brief

## Design Goal
...

## Player Experience
...

## Core Interaction
...

## Success Criteria
...

## Non-Goals
...

## Open Questions
...
```

## After Updating Design

When a `Dora` task meaningfully changes the project discussion, add or update a
matching worklog entry separately when appropriate.

Do not treat handoff notes as a permanent section inside every design document.
