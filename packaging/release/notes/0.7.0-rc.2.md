# Loomle 0.7.0-rc.2

This release candidate improves Loomle's control-plane visibility and makes
Blueprint Graph Pin identity handling safer.

## Supported environment

- Unreal Engine 5.7
- macOS on Apple Silicon
- Windows on x64
- Engine-plugin installation

Remove or move any old project-local `Plugins/LoomleBridge` copy before using
0.7. Unreal gives a same-named project plugin precedence over the engine
plugin.

## Changes since rc.1

- Added the read-only `status` MCP call for Client version, update
  availability, session binding, and Bridge health.
- Added a bounded, cached release-manifest check with exact target download
  URLs and SHA-256 values.
- Added concise agent update guidance, including Windows-only Client process
  shutdown instructions before replacing the plugin.
- Scoped Graph Pin identity resolution to the owning Node and Graph, while
  failing closed when a Pin selector is genuinely ambiguous.
- Expanded packaged Client smoke coverage to include the complete six-tool
  public surface.

## Current limits

- Runtime UI observation, animation/navigation editing, automatic Blueprint
  layout, and cross-Graph execution expansion remain outside this candidate.

## macOS security notice

This release candidate is not yet signed with a Loomle Developer ID certificate
or notarized by Apple. macOS Gatekeeper may therefore require the user to
explicitly approve the downloaded plugin or Client. Stable Loomle releases will
remain blocked until signing and notarization are part of the exact artifact
path tested before publication.

## Windows security notice

This release candidate is not Authenticode-signed. Windows may therefore show a
trust or SmartScreen warning for the downloaded plugin or Client. Stable Loomle
release policy will be finalized before signed Windows artifacts are published.
